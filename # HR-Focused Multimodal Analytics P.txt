# HR-Focused Multimodal Analytics Platform: Business Plan

## Executive Summary

### Platform Overview
Our HR Analytics Platform serves as a groundbreaking solution that transforms workplace management through advanced multimodal data analytics. By integrating real-time data sourced from webcams, human interface devices (HID), and activity logs, we provide comprehensive insights into workplace dynamics, employee well-being, and organisational efficiency. The platform acts as a central nervous system for HR operations, enabling data-driven decision-making while ensuring the highest standards of privacy and security are maintained.

### Value Proposition
- **For HR Teams:** Improved workforce management, automated monitoring, and predictive insights that reduce administrative burdens and facilitate proactive interventions.
- **For Employees:** Enhanced workplace well-being through personalised ergonomic guidance, stress management strategies, and health optimisation techniques.
- **For Organisations:** Increased productivity, reduced healthcare expenses, and lower absenteeism through data-informed optimisation of workplace environments.
- **For Stakeholders:** Tangible return on investment through a decrease in workplace injuries, enhanced employee retention rates, and improved operational efficiency.

### Key Differentiators
1. **Privacy-First Architecture**
   - Options for self-hosted deployment
   - Comprehensive end-to-end encryption
   - Detailed privacy controls
   - Full data sovereignty

2. **Comprehensive Analytics Suite**
   - Integration of real-time multimodal data
   - Advanced predictive modelling capabilities
   - Customisable dashboards for user-specific needs
   - Generation of actionable insights

3. **Enterprise-Grade Scalability**
   - Modular system design for flexibility
   - Capabilities for seamless integration
   - Custom deployment options tailored to client requirements
   - A robust security framework to protect data integrity

# Function 3: Track Activity (Keyboard & Mouse)
# ----------------------------------
def activity_tracker(duration=10):
    """
    Tracks keyboard press events and mouse click events for a specified duration.
    Logs events along with their timestamps relative to the start of tracking.
    
    Parameters:
        duration (int): Time period (in seconds) for tracking activity.
        
    Returns:
        events (list): Logged events.
    """
    start_time = time.time()
    events = []

    def on_press(key):
        timestamp = time.time() - start_time
        events.append({
            "event": "key_press",
            "key": str(key),
            "time": round(timestamp, 2)
        })

    def on_click(x, y, button, pressed):
        timestamp = time.time() - start_time
        events.append({
            "event": "mouse_click",
            "button": str(button),
            "pressed": pressed,
            "position": (x, y),
            "time": round(timestamp, 2)
        })

    # Create listeners for keyboard and mouse
    keyboard_listener = keyboard.Listener(on_press=on_press)
    mouse_listener = mouse.Listener(on_click=on_click)

    keyboard_listener.start()
    mouse_listener.start()

    print(f"Activity tracking started for {duration} seconds...")
    time.sleep(duration)

    keyboard_listener.stop()
    mouse_listener.stop()

    print("Activity tracking complete. Logged events:")
    for event in events:
        print(event)
    return events

# Main Execution: Running the Modules
# ----------------------------------
if __name__ == "__main__":
    # Using threading to run activity tracking concurrently can simulate capturing activity
    # while performing other actions. Adjust durations as needed.
    
    # Start the activity tracker in a separate thread (e.g. for 15 seconds)
    tracker_thread = threading.Thread(target=activity_tracker, args=(15,))
    tracker_thread.start()
